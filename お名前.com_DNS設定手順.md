# お名前.com で eli-co.jp を GitHub Pages に向ける手順

GitHub Pages のカスタムドメインで **eli-co.jp**（ルートドメイン）を使うための、お名前.com での DNS 設定手順です。

---

## 前提

- お名前.com で **eli-co.jp** を取得済みであること
- GitHub の Pages 設定で Custom domain に **eli-co.jp** を入力済みであること

---

## Step 1: お名前.com Navi にログイン

1. ブラウザで **https://navi.onamae.com/** を開く
2. 会員ID（お名前ID）とパスワードでログインする

---

## Step 2: ドメイン一覧を開く

1. 左メニュー（またはトップメニュー）の **「ドメイン」** をクリック
2. **「ドメイン一覧」** を開く
3. 一覧から **eli-co.jp** を探し、その行の **「DNS設定」** または **「DNSレコード」** の **「変更」**／**「設定する」** をクリック

（画面によっては「ドメイン設定」→「DNS設定」などの名前の場合があります）

---

## Step 3: DNSレコード設定を利用する

1. **「DNSレコード設定を利用する」** の **「設定する」** をクリック
2. すでに「DNSレコード設定を利用する」になっている場合は、そのまま **「変更」** または **「レコード設定」** をクリックして設定画面へ進む

**注意**: 現在「お名前.comのネームサーバー」（例: ns1.dnsv.jp など）以外を使っている場合は、先にネームサーバーをお名前.com のものに戻す必要があります。不明な場合はお名前.com サポートに確認してください。

---

## Step 4: Aレコードを 4 件追加する（eli-co.jp を GitHub Pages に向ける）

### 4-0. 「レコード追加」タブを開く

1. DNSレコード設定の画面で、上にある **「レコード追加」** タブをクリックする
2. 「登録済レコード」ではなく **「レコード追加」** のタブが選ばれた状態になる（入力欄やプルダウンが表示される）

---

### 4-1. 1 本目の A レコードを追加

「レコード追加」のフォームで、次のように入力する。

| 項目名 | 入力する内容 |
|--------|----------------|
| **ホスト名** | 何も入力しない（空欄のまま）。または `@` と入力する |
| **TYPE（種別）** | プルダウンから **「A」** を選ぶ |
| **TTL** | 空欄のままでよい（または 3600 など） |
| **VALUE（値・IPアドレス）** | `185.199.108.153` と入力する |

入力後、**「追加」** ボタンをクリックする。  
→ 1 本目が「登録予定」の一覧に表示される。

---

### 4-2. 2 本目の A レコードを追加

もう一度「レコード追加」のフォームに次のように入力する。

| 項目名 | 入力する内容 |
|--------|----------------|
| **ホスト名** | 空欄のまま（または `@`） |
| **TYPE** | **A** |
| **VALUE** | `185.199.109.153` |

**「追加」** をクリックする。

---

### 4-3. 3 本目の A レコードを追加

同様にフォームに入力する。

| 項目名 | 入力する内容 |
|--------|----------------|
| **ホスト名** | 空欄のまま（または `@`） |
| **TYPE** | **A** |
| **VALUE** | `185.199.110.153` |

**「追加」** をクリックする。

---

### 4-4. 4 本目の A レコードを追加

最後の 1 本を入力する。

| 項目名 | 入力する内容 |
|--------|----------------|
| **ホスト名** | 空欄のまま（または `@`） |
| **TYPE** | **A** |
| **VALUE** | `185.199.111.153` |

**「追加」** をクリックする。

---

### 4-5. 4 本とも追加されたか確認する

1. **「登録済レコード」** タブ（または「登録予定」の一覧）を開く
2. **TYPE が A** で、**VALUE が上記 4 つの IP** のレコードが **4 本** 並んでいることを確認する
3. 足りない場合は「レコード追加」タブに戻り、同じ要領でもう 1 本ずつ追加する

---

## Step 5: （任意）www.eli-co.jp も同じサイトにする場合

**www.eli-co.jp** でアクセスしたときも同じ GitHub Pages のサイトを表示したい場合は、**CNAME レコード** を 1 件追加します。

| 項目 | 入力値 |
|------|--------|
| **ホスト名** | `www` |
| **TYPE** | `CNAME` |
| **VALUE** | `eliymkipad2023-ship-it.github.io.` |

※ VALUE の **末尾のドット（.）** を忘れずに入力する。

→ **「追加」** をクリック

---

## Step 6: 設定を反映させる

1. 画面を下にスクロールする
2. **「転送用のネームサーバーに変更する」** にチェックが入っていることを確認する（DNSレコード設定を使う場合は通常ここにチェックを入れてから反映します）
3. **「設定する」** または **「保存」** をクリックする
4. 確認画面が出たら、内容を確認して確定する

---

## Step 7: 反映を待つ

- DNS の反映には **数分〜最大 48 時間** かかることがあります（多くの場合は 10〜30 分程度）
- 待ったあと、GitHub の **Settings → Pages → Custom domain** の **「Check again」** をクリックして、エラーが解消されているか確認する
- 問題なければ **「Enforce HTTPS」** にチェックを入れる

---

## うまくいかないときの確認

1. **ネームサーバー**: eli-co.jp のネームサーバーが **お名前.com の DNS**（例: ns1.dnsv.jp, ns2.dnsv.jp など）になっているか
2. **A レコード**: 4 本とも **ホスト名が空欄または @** で、**VALUE の IP が 4 つとも正確** か
3. **既存レコード**: 同じホスト名の **古い A レコード** や **CNAME** が残っていないか（重複や競合があると GitHub に解決されません）
4. **時間**: 設定後、最低 10〜15 分は待ってから GitHub で「Check again」する

---

## まとめ（やること一覧）

1. お名前.com Navi にログイン
2. ドメイン一覧 → **eli-co.jp** → DNS設定／DNSレコード設定を開く
3. **A レコードを 4 本追加**（ホスト名: 空欄 or @、VALUE: 上記 4 つの IP）
4. （任意）**CNAME 1 本追加**（ホスト名: www、VALUE: eliymkipad2023-ship-it.github.io.）
5. 「転送用のネームサーバーに変更する」にチェック → 設定する
6. 数分〜最大 48 時間待ってから GitHub で「Check again」
